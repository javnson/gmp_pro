# GMP MATLAB/Simulink Library (`slib`)

**English** | [简体中文](readme_cn.md)

`slib` is the source and release builder for GMP's MATLAB/Simulink integration. It provides reusable plant/peripheral blocks, fixed-point-compatible controller helpers, the UDP software-in-the-loop (SIL) bridge, standard power-stage models, and MATLAB initialization helpers generated by SDPE.

## Requirements and support boundary

- MATLAB and Simulink R2022b or later are required by `upgrade_gmp_simulink_lib.m`.
- Existing power-stage models require Simscape Electrical Specialized Power Systems and its `powerlib` library.
- The current installer deliberately rejects R2026a and later because Specialized Power Systems is no longer available there. Use R2025b or earlier for the current models.
- Linux SIL is a formally supported direction, but its current implementation is incomplete: a Linux x64 `GMP_SIL_Core` is present while no Linux UDP-helper binary is currently shipped. Keep the current Linux code unchanged until that path is completed and validated.
- A MATLAB Release can only use compatible SLX and MEX artifacts. Do not assume that a generated directory for one Release is valid for another.

The installation script tests whether `powerlib` can actually be loaded. A product appearing in MATLAB metadata is not sufficient if its files or license are unavailable.

## Directory map

| Path | Role | Maintenance rule |
| --- | --- | --- |
| `simulink_lib_src/*.slx` | Authoritative Simulink library models | Edit these models. |
| `simulink_lib_src/src/` | MATLAB helpers and committed MEX binaries | Edit/test helpers here; rebuild a MEX binary from its native source project. |
| `simulink_lib_src/tests/` | MATLAB regression tests | Add tests for helper behavior and path-independent initialization. |
| `simulink_lib_src/icon/` | Library Browser icons | Keep model icon references relative. |
| `simulink_lib_src/avatars/` | STEP/SolidWorks visualization assets | Treat as source assets. |
| `install_path/<Release>/` | Generated, installable library for one MATLAB Release | Never edit directly; regenerate it. |
| `tools/` | S-Function Builder controller examples and build metadata | Keep generated MEX files synchronized with their C++ wrappers. |
| `future/` | Experimental mechanical/electrical models | Not part of the stable installed library contract. |

`install_path/R2022b` and `install_path/R2024b` may exist in a checkout, but all release directories are treated as generated output by the repository `.gitignore`.

## Installed libraries

`slblocks.m` registers `gmp_simulink_utilities` as **GMP Utilities Library** in the Simulink Library Browser. That top-level library leads to the following generated libraries:

| Library | Purpose |
| --- | --- |
| `gmp_simulink_utilities` | Top-level navigation for GMP controller, plant, SIL, peripheral, and numeric utilities. |
| `gmp_fp_utilities` | GMP floating-point compatibility functions, transforms, PWM helpers, and measurement helpers. |
| `gmp_peripheral_utilities` | Virtual ADC, PWM, sensor, sampling, and peripheral behavior. |
| `gmp_sil_core_pack` | GMP SIL controller panels, channel packing, trigger handling, and UDP-facing core blocks. |
| `gmp_std_model_pck` | Standard converter, machine, sensor, and power-stage models. |
| `gmp_component_model` | S-Function wrappers for selected CTL components. |

The authoritative source model names end in `_src.slx`. `upgrade_gmp_simulink_lib` saves release-compatible copies without `_src` under `install_path/<Release>`.

## Install, upgrade, and uninstall

First install GMP so that `GMP_PRO_LOCATION` points to the repository root. Then run this from MATLAB:

```matlab
run(fullfile(getenv('GMP_PRO_LOCATION'), ...
    'slib', 'install_gmp_simulink_lib.m'));
```

The installer:

1. validates `powerlib` and the MATLAB Release;
2. changes temporarily to `slib` so generation is independent of the caller's working directory;
3. runs `upgrade_gmp_simulink_lib` to build `install_path/<Release>`;
4. copies MATLAB helpers, MEX files, icons, and avatar assets;
5. adds the generated directory and its `src` directory to the MATLAB path;
6. saves the path and refreshes Simulink customizations.

To remove the current Release's generated library and MATLAB paths:

```matlab
run(fullfile(getenv('GMP_PRO_LOCATION'), ...
    'slib', 'uninstall_gmp_simulink_lib.m'));
```

`reg_path_gmp_simulink_lib.m` only re-adds an already-generated release directory. It does not validate dependencies or regenerate models.

Uninstallation removes the generated paths, persists that removal with `savepath`, and then deletes the current Release directory.

## SIL architecture

A typical GMP SIL suite has two cooperating processes:

```text
Simulink power stage and virtual peripherals
        | ADC codes, time, panel/digital inputs
        v
GMP_SIL_Core / MEX_UDP_Helper  <--- UDP --->  Windows controller executable
        ^                                      (same shared suite/src controller)
        | Enable, PWM compare, DAC/monitor data
        +--------------------------------------
```

The native controller uses `csp/windows_simulink` and `SPECIFY_PC_ENVIRONMENT`. Its main loop receives a packed `gmp_pc_simulink_rx_buffer_t`, calls `gmp_base_ctl_step()`, and returns a packed `gmp_pc_simulink_tx_buffer_t`. Each suite's `xplt.ctl_interface.h` owns the exact ADC/PWM channel mapping; the Simulink model and controller must agree on ordering, type, size, alignment, scaling, polarity, and UDP ports.

Do not change a model channel order to hide a target mapping error. Record the contract in the suite's simulation README and fix the model plus `xplt` mapping together.

`mdl_gmp_simulink_connection` validates a plain IPv4 address and produces the four-port vector used for message TX/RX and command TX/RX. `sl_udp_server_init_cb` releases stale sockets, configures the MEX helper, links the network, and sends `start`; `sl_udp_server_stop_cb` sends `stop` and releases the network.

## SDPE model initialization

Current suites use two generated SDPE layers:

```text
<suite>/sdpe_general/*_matlab_init.m
<suite>/project/<target>/sdpe_mgr/*_matlab_init.m
```

Use:

```matlab
gmp_run_model_sdpe_init(bdroot)
```

The helper resolves paths from the saved model file, requires exactly one `*_matlab_init.m` in each layer, and runs the general layer before the target layer in the MATLAB base workspace. It intentionally rejects a missing or ambiguous layer.

Before opening or running a model after an SDPE JSON change, regenerate both layers:

```bat
sdpe_general\sdpe_generate.bat
project\simulate\sdpe_mgr\sdpe_generate.bat
```

Avoid absolute checkout paths in `PreLoadFcn`, `PostLoadFcn`, or `InitFcn`. Resolve paths from the model filename or call `gmp_run_model_sdpe_init`.

## Developing the library

1. Edit the relevant `_src.slx` model or a helper under `simulink_lib_src/src`.
2. Keep block callbacks and mask code independent of the current working directory.
3. Add or update a regression test under `simulink_lib_src/tests`.
4. Run the MATLAB tests from the repository checkout:

   ```matlab
   addpath(fullfile(getenv('GMP_PRO_LOCATION'), 'slib', ...
       'simulink_lib_src', 'src'));
   results = runtests(fullfile(getenv('GMP_PRO_LOCATION'), 'slib', ...
       'simulink_lib_src', 'tests'));
   assert(all([results.Passed]));
   ```

5. Run `install_gmp_simulink_lib` to regenerate the current Release.
6. Open the Library Browser and verify block links, masks, icons, and required libraries.
7. Validate at least one suite simulation that exercises the changed block or transport contract.

For S-Function Builder components under `tools/`, preserve the `.cpp`, wrapper, `SFB__*.mat`, `rtwmakecfg.m`, and compiled MEX relationship. Do not treat a committed MEX file as editable source.

## Troubleshooting

| Symptom | Check |
| --- | --- |
| `powerlib` is missing or cannot load | Install/license Simscape Electrical Specialized Power Systems and use R2025b or earlier. |
| GMP library is absent from Library Browser | Confirm the generated Release path and `src` are on the MATLAB path, then run `sl_refresh_customizations`. |
| An `_src.slx` library opens but the installed library is stale | Run `install_gmp_simulink_lib`; do not patch `install_path` manually. |
| `GMP_SIL_Core` or `MEX_UDP_Helper` is not found | Confirm platform-compatible MEX files are on the generated `src` path. |
| UDP communication times out | Compare `network.json`, the model mask, process startup order, firewall rules, and all four ports. Release stale sockets before retrying. |
| SDPE variables are undefined | Generate both SDPE layers and verify the model is saved in `<suite>/project/<target>`. |
| SIL results have incorrect sign or channel | Compare the model wiring with the target's `xplt.ctl_interface.h`; also verify ADC bias/gain and PWM polarity. |

The installed helper reports its current library identity through `get_gmp_slib_version()`; the floating-point compatibility API has a separate `gmp_float_macros_version()`.
