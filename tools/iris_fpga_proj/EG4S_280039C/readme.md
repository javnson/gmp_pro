# DSP 与 FPGA 通信及内存映射设计说明

## 1. 当前 FPGA 内存映射结构 (Memory Mapping)

根据 `top_soc` 模块中的总线路由定义，FPGA 内部逻辑通过一个 **256 位的虚拟总线** 被分割成了 **16 个 16 位（1 Word）的寄存器 (R0 ~ R15)**。每个寄存器的地址及映射关系如下表所示：  

| **寄存器名** | **伪地址 (Hex)** | **数据总线切片 (Bit)** | **访问属性** | **硬件对应与功能说明  未知**                                 |
| ------------ | ---------------- | ---------------------- | ------------ | ------------------------------------------------------------ |
| **R0**       | `0x00`           | `[15:0]`               | 读/写        | 保留配置寄存器（写啥读啥，回环测试用）                       |
| **R1**       | `0x01`           | `[31:16]`              | 读/写        | GPIO 输出寄存器。低 4 位 `[19:16]` 映射到板载 **LED**        |
| **R2**       | `0x02`           | `[47:32]`              | 只读         | GPIO 输入寄存器。高 12 位补 0，低 4 位实时反映**拨码开关 (Encoder)** 状态 |
| **R3**       | `0x03`           | `[63:48]`              | 读/写        | **ADC 控制寄存器**。低 8 位 `[55:48]` 作为 ADS8688 的通道屏蔽掩码 (`channel_mask`) |
| **R4**       | `0x04`           | `[79:64]`              | 读/写        | **DAC 通道 1** 目标输出数据                                  |
| **R5**       | `0x05`           | `[95:80]`              | 读/写        | **DAC 通道 2** 目标输出数据                                  |
| **R6**       | `0x06`           | `[111:96]`             | 读/写        | **DAC 通道 3** 目标输出数据                                  |
| **R7**       | `0x07`           | `[127:112]`            | 读/写        | **DAC 通道 4** 目标输出数据                                  |
| **R8**       | `0x08`           | `[143:128]`            | 只读         | **ADC 通道 0** 的实时采集结果缓存                            |
| **R9**       | `0x09`           | `[159:144]`            | 只读         | **ADC 通道 1** 的实时采集结果缓存                            |
| **R10**      | `0x0A`           | `[175:160]`            | 只读         | **ADC 通道 2** 的实时采集结果缓存                            |
| **R11**      | `0x0B`           | `[191:176]`            | 只读         | **ADC 通道 3** 的实时采集结果缓存                            |
| **R12**      | `0x0C`           | `[207:192]`            | 只读         | **ADC 通道 4** 的实时采集结果缓存                            |
| **R13**      | `0x0D`           | `[223:208]`            | 只读         | **ADC 通道 5** 的实时采集结果缓存                            |
| **R14**      | `0x0E`           | `[239:224]`            | 只读         | **ADC 通道 6** 的实时采集结果缓存                            |
| **R15**      | `0x0F`           | `[255:240]`            | 只读         | **ADC 通道 7** 的实时采集结果缓存                            |

## 2. DSP 如何访问这些寄存器

DSP 通过标准 SPI 协议（结合片选信号 `spi_csn`）对 FPGA 进行同步读写。根据你的测试代码 `fpga_test_task`，其内部蕴含了以下**核心访问逻辑与硬件联动机制**：  

1. **写寄存器 (`SPI_writeReg`) 触发机制：**
   - 当 DSP 调用 `SPI_writeReg(addr, data)` 时，FPGA 会在片选拉高后内部产生一个 **`flush_pulse`（同步脉冲信号）**。  
   - **LED 闪烁控制：** 代码中通过对 `0x01` (R1) 周期性交替写入 `0x0003` 和 `0x0000`，控制对应 LED 的亮灭。  
   - **ADC 间接触发：** 你的测试程序向 `0x05`, `0x06`, `0x07` (R5~R7) 连续写入了数据。由于这些写操作会在 FPGA 内部产生 `flush_pulse`，而该脉冲直接连接到了 ADC 模块 `ads8688_masked_ctrl` 的 **`trigger` 输入端**。因此，**每次写任意寄存器（如 R5, R6, R7 等），都会实质上向 ADC 发出一次采集命令。**  
2. **读寄存器 (`SPI_readReg`) 机制：**
   - 当 DSP 想要获取外部硬件状态（如数据或通道值）时，调用 `SPI_readReg(addr)`。  
   - 在代码中，DSP 随后读取了 `0x08` (R8) 寄存器的值，将其暂存到 `adc_result` 变量中，从而成功拿到了 **ADC 通道 0** 缓存的最新转换结果。之后又将其写入 `0x04` (R4)，改变了 DAC 通道 1 的物理输出。  

## 3. 如果需要扩展，应该如何做？

当前的寄存器映射通过 256 位宽度的 `spi_in_wires` 和 `spi_out_regs` 完成，已经全部用满（$16 \text{ registers} \times 16 \text{ bits} = 256 \text{ bits}$）。如果未来需要增加新的控制寄存器或状态只读寄存器，请按照以下步骤调整 FPGA 代码：  

### 第一步：扩展位宽与子模块改动

1. 打开 `spi_slave_regs_easy` 模块，将其内部对应的 `in_wires` 与 `out_regs` 接口位宽增加（例如，若想扩展至 32 个寄存器，位宽需要改到 **512 位**）。  

2. 同步在 `top_soc` 模块中，将内部线网定义修改为：

   Verilog

   ```verilog
   wire [511:0] spi_in_wires;
   wire [511:0] spi_out_regs;
   ```

### 第二步：定义新的内存路由

在 `3. 内存映射核心路由` 区域，继续按照 16 位累加原则往后追加路由配置。  

- **添加扩展写/读回环寄存器（例如 R16, 地址 0x10）：**

  Verilog

  ```verilog
  // R16 (0x10): 新增控制寄存器
  assign spi_in_wires[271:256] = spi_out_regs[271:256];
  assign my_new_hardware_ctrl  = spi_out_regs[271:256]; // 连线到你的新逻辑
  ```

- **添加扩展只读状态寄存器（例如 R17, 地址 0x11）：**

  Verilog

  ```c
  // R17 (0x11): 新增只读寄存器（如外部故障引脚状态）
  assign spi_in_wires[287:272] = {15'h000, error_flag_pin};
  ```

### 第三步：DSP 端同步升级

在 DSP 的驱动或头文件中，新增对应的地址宏定义，随后直接在控制任务中调用即可：

C

```c
#define FPGA_REG_NEW_CTRL  0x10
#define FPGA_REG_NEW_STAT  0x11

// DSP 即可直接访问
SPI_writeReg(FPGA_REG_NEW_CTRL, 0x1234);
uint16_t status = SPI_readReg(FPGA_REG_NEW_STAT);
```
